# Product Agent Review

Status: COMPLETE

## Findings

Claude Desktop-first is the right V1. The wedge is narrow, painful, local, and concrete: "my MCP server does not show up in Claude Desktop on Windows."

## First-Dollar User

AI builders, consultants, and technical operators who repeatedly set up MCP servers for themselves, teams, or clients.

## Cuts

- Do not meaningfully support macOS/Linux in V1.
- Cut broad "common MCP issues" unless each maps to direct evidence.
- Keep `--fix` tiny.
- Do not infer missing secrets beyond empty or missing config values.
- Do not let Agent Bridge artifacts outrun scan quality.

## Must Be Excellent

- Secret masking.
- Plain-English diagnosis with exact evidence.
- Windows behavior around `%APPDATA%`, PowerShell users, `npx`, `cmd /c`, and PATH checks.
- Useful no-config state.

## Landing Page Promise

Fix Claude Desktop MCP setup problems locally, without uploading your config or secrets.
